# CLAUDE.md — PITS-MRAS

Project-specific guidance for Claude Code in this repo. Keep terse; details live in
`docs/` and `CHANGELOG.md`.

PITS-MRAS = Physics-Informed Time-Series Model-Reference Adaptive Systems: PINNs
(port-Hamiltonian decoder) + time-series DL (LSTM + attention) + MRAS, unified with
RL/optimal control. Python 3.10+ (CI: 3.10–3.12), src-layout (`src/pits_mras/`).

## Gates — run before claiming done / committing

```bash
python -m pytest -q
ruff check .
ruff format --check .
mypy src/pits_mras --ignore-missing-imports
```

After any change to the source structure (new/removed files, exports), regenerate the
dependency graph and reconcile the graph-backed docs:

```bash
python tools/create-dependency-graph/create_dependency_graph.py --include-tests
```

Note: the suite runs ~5 min — the un-skipped tight H∞ min-max recovery test does an
8000-iteration optimization (deterministic, ~3 min). That's expected, not a hang.

## Toolchain

- **Ruff is pinned exactly (`ruff==0.8.1`)** in `requirements-dev.txt` and the
  `setup.py` dev extras. Do NOT loosen the pin — an unpinned ruff floats to a newer
  formatter and red-lights CI on format drift. Ruff replaces flake8/black/isort.
- mypy for typing. Line length 100. `jaxtyping` is dev-only (TYPE_CHECKING).

## Documentation conventions

- **`docs/architecture/*` and the design docs are version/date-free, present-tense
  SNAPSHOTS of the current design.** No version numbers, dates, or "added/changed/
  resolved in vX" prose in their bodies. Version/date history lives in `CHANGELOG.md`
  and the CDG-generated reports (`DEPENDENCY_GRAPH.md`, `TEST_COVERAGE.md`,
  `unused-analysis.md`, the `*.json/yaml`). The CDG tool no longer stamps version/date.
- The CDG-generated reports are regenerated by the tool — never hand-edit them.
- The root-level `docs/*.md` design specs (Blueprint / Implementation Plan / PCML
  Addendum / Physics-Informed spec / PRD / validation report) and everything under
  `docs/superpowers/` are **point-in-time records — do not rewrite them to "current".**
  `docs/architecture/ARCHITECTURE.md` is the authoritative current layout/identity map.

## Workflow & commits

- TDD (failing test first); atomic commits (one logical change); no `--no-verify`
  / `--no-gpg-sign` / hook-skipping. Root-cause fixes, not bypasses.
- Per substantive item: bump version (`setup.py` + `src/pits_mras/__init__.py`) and
  tag (`vX.Y.Z`); rename CHANGELOG `[Unreleased]` → the version. Feature → minor,
  fix/refactor → patch. Push directly to `master` (no PR flow); confirm CI green.
- Commit messages end with a `Co-Authored-By:` footer (set by the harness).

## Gotchas

- **`.gitignore` `data/` rule:** `src/pits_mras/data/` is a real source package; the
  generic `data/` ignore would silently un-track it. An exception keeps it tracked —
  after adding ANY new module whose dir name is a common ignore token (`data/`,
  `logs/`, `build/`), verify with `git ls-files <path>` / `git check-ignore <file>`.
  A green local pytest does NOT prove the files are committed.
- CRLF→LF warnings on `git add` are expected (`.gitattributes` normalizes to LF).

## Honesty caveats (keep in docs/claims)

- Example plants are illustrative nonlinear surrogates, **not hardware-validated**;
  performance figures in docs are **design targets**, not measured hardware results.
- Several capabilities are additive/opt-in and verified structurally (finiteness,
  shapes, oracle-recovery on linear surrogates) — not as end-to-end trained systems.
