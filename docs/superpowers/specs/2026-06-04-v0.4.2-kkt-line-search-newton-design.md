# KKT projection: damped / line-search Newton (design spec)

**Goal:** Make `KKTProjectionLayer`'s forward Newton solve robust to overshoot by
adding a backtracking line search on the residual, so the projection converges on
stiff / high-curvature constraints where the full Newton step diverges. Release
v0.4.2.

**Date:** 2026-06-04
**Sub-project:** 3 of the remaining v0.4.0 set. Resolves carried-forward gap #1
(only the *signal* half of the original debt #2 shipped in v0.3.2; this adds the
robustness half).

## Problem

`KKTProjectionLayer.forward` takes an **undamped full Newton step**
(`z = z - newton_step * delta`, `newton_step=1.0`). On smooth, well-conditioned
constraints (the current heat-eq / mechanical-DAE / linear-equality cases) this
converges. On a high-curvature constraint it **overshoots and diverges** —
verified: a single equality `h(y) = atan(8·y) = 0` projected from `y_hat ∈
{2,3,5}` blows the residual up to ~2.3e6 in 20 iterations (no convergence). The
v0.3.2 work only added the convergence *signal* (`last_converged`/`last_residual`
+ warning); the iterate is still taken at a non-stationary point.

## Design

Add a **backtracking line search** to the forward Newton loop. The Newton
direction `delta` (Gauss-Newton, `(J + reg·I)⁻¹ F`) is a descent direction for
the merit `‖F‖`; backtracking finds a step length that actually decreases it.

- New `__init__` params: `use_line_search: bool = True`,
  `line_search_max_halvings: int = 10`.
- Per Newton iteration (when `use_line_search`): start `α = newton_step`; form the
  trial `z' = z − α·delta`; evaluate the residual `‖F(z')‖_∞`; accept the first
  `α` (halving each time, up to `line_search_max_halvings`) for which
  `‖F(z')‖_∞ < ‖F(z)‖_∞`. If no halving decreases the residual, accept the
  smallest-`α` trial (still moves; convergence is then reported honestly via
  `last_converged`). With `use_line_search=False`, keep the exact old full-step.
- **Loop restructure (carefully):** compute `(c, g, Jc, Jg)` + `F` once before the
  loop and reuse each accepted trial's constraints/Jacobian for the next
  iteration — so the happy path costs ~1 `_constraints_and_jac` per iteration
  (same as today); only actual backtracks add evaluations.
- **Differentiability is unchanged.** The implicit-function one-step
  (`z_out = z_star − (J+reg·I)⁻¹ F_live(y_hat, z_star)`) is untouched — it depends
  only on the converged iterate `z_star`, not on how it was found. `last_cj`
  always holds the constraints/Jacobian at `z_star`, so the IFT reuse stays valid.

### Behavior preservation

On the existing (smooth) problems the full step already decreases the residual, so
the line search accepts `α = newton_step` on the first try → identical path →
identical `z_star` → identical output. All existing PCML tests stay green and
output-equivalent. The change only adds value when the full step would overshoot.
