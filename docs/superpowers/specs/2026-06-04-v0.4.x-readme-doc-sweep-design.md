   keeping the honest caveat that the example plants are toy linear surrogates and
   the framework is research-grade.
3. **Prerequisites** — `Python 3.8+` → `3.10+`.
4. **Quick Start** — the current snippet is **non-functional**: `MRASController(...,
   adaptation_rate=0.01)`, `initialize_controller`, `closed_loop_training`,
   `inference_realtime`, and the `PITNN(input_dim=..., hidden_dim=..., output_dim=...,
   lstm_layers=..., attention_heads=...)` kwargs do not exist. Replace with the real
   API (grounded in current signatures):

     RealtimeInferenceEngine, pretrain_pitnn, cotraining_loop`; configs from
     `pits_mras.config import PITSMRASConfig, NetworkConfig, PhysicsConfig`;
   - `cfg = PITSMRASConfig(); cfg.network = NetworkConfig(input_dim=2, hidden_dim=64,
     output_dim=2, lstm_layers=1, attention_heads=2, embedding_dim=16);
     cfg.physics = PhysicsConfig(n_generalized_coords=1)`;
   - `pitnn = PITNN(cfg.network, cfg.physics)`;
   - `ref = LinearReferenceModel(A_m, B_m, C_m, Q, R)` (Hurwitz `A_m`);
   - `controller = MRASController(reference_model=ref, state_dim=2, control_dim=1,
     ref_dim=1, plant_dim=2, use_safety_filter=True)`;

   - Phase 2 `cotraining_loop(pitnn, controller, ref, cfg, n_episodes=5, n_steps=50)`;
