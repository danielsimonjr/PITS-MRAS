# ParallelInferenceEngine hardening (design spec)

**Goal:** Take the three-thread `ParallelInferenceEngine` from an honest skeleton
(no-op adaptation, silent thread death) to a hardened deployment scaffold with a
**real double-buffered critic update** and **no silent thread failures**. Release
v0.4.4.

**Date:** 2026-06-04
**Sub-project:** 5 of the remaining v0.4.0 set (the last autonomous one; H∞ stops
for its own brainstorm).

## Current gaps (honest skeleton)

1. **Silent thread death.** Each loop body runs unguarded in a daemon thread; if
   `engine.step` (or anything) raises, the thread dies silently and the system
   keeps "running" with a dead control/adaptation thread, undetected.
2. **No-op adaptation.** `_adaptation_loop` deepcopies the critic, does *nothing*
   to the copy, and swaps it back — the double-buffer structure is wired but the
   update is a placeholder.

## Design

### 1. Exception capture (fail-fast, no silent death)

A `_run_guarded(name, body)` wrapper runs each loop; on any exception it records
the first failure (`self._error`, `self._failed_thread`) and sets `_shutdown`
(so a failure in one thread stops the whole engine rather than half-running).
Add `error` (property → the stored exception or `None`) and `check()` (re-raises
the stored exception if any). `is_alive()` / `stop()` unchanged.

### 2. Real double-buffered adaptation

- **Control → adaptation handoff:** the control thread appends the per-cycle
  `(e, u_safe)` (detached, 1-D) to a bounded `deque(maxlen=irl_window+1)` under a
  `_window_lock`.
- **`_adaptation_update() -> bool`** (factored out so it's unit-testable without
  threads): snapshot the window under the lock; if it holds fewer than
  `irl_window+1` samples, return `False` (nothing to fit yet). Otherwise:
  `deepcopy` the live critic, stack the window into `e_win [1, W+1, n]` /
  `u_win [1, W+1, m]`, take **one** Adam gradient step of
  `IRLBellmanLoss(ref_model.Q, ref_model.R)` on the copy (grad-clip 1.0, as in
  `cotraining_loop`), then **atomically swap** the copy in under `_critic_lock`,
  increment `state.adaptation_swaps`, return `True`. The heavy work (deepcopy +
  gradient step) happens **off** `_critic_lock`; only the pointer swap is locked,
  so the control thread is never exposed to a half-updated critic and never
  blocks on the adaptation compute.
- `_adaptation_loop` just calls `_adaptation_update()` each cycle (guarded).
- `ControllerState` gains `adaptation_swaps: int = 0`.
- Constructor gains `irl_window: int = 8`, `adapt_lr: float = 1e-3`.

### Thread-safety

The control thread holds `_critic_lock` for its `engine.step` (so a step sees one
consistent critic); the adaptation thread does its compute on an isolated
`deepcopy` and takes `_critic_lock` only for the atomic reassignment. The control
thread's inference is `@torch.no_grad`; the adaptation autograd is on the copy —
no shared-tensor autograd contention. Window access is under `_window_lock`.

### Still scaffold (documented honestly)

`x_p` / `r` remain fixed per cycle (no live sensor I/O — a real deployment reads
sensors); the scheduler is cooperative `Event.wait(period)`, not hard-real-time.
The docstring states this plainly.

## Testing (TDD)

In `tests/test_inference.py`:
1. **`test_parallel_adaptation_update_runs_irl_step`** (deterministic, no
   threads): build the engine, perturb the critic off `P_opt`, pre-fill the
   window with `irl_window+1` `(e, u)` samples, call `_adaptation_update()` →
   returns `True`, `state.adaptation_swaps == 1`, and the live critic's `W_c`
   changed (adaptation actually updated it). Calling with an empty window →
   `False`, no swap.
2. **`test_parallel_engine_captures_thread_exception`**: a stub engine whose
   `step()` raises → `start()`, `wait`, `stop()` → `par.error is not None`,
   `not par.is_alive()`, and `check()` re-raises.
3. **`test_parallel_engine_adapts_during_run`** (threaded, structural): run the
   real engine ~0.3 s → `state.step_count > 0` and `state.adaptation_swaps > 0`
   (the window fills and the adaptation ticks); all shared metrics finite; joins
   cleanly. Generous wait to avoid timing flakiness; the *correctness* of the
   update is pinned by test 1, this only checks the loop wiring.
4. Existing parallel tests (start/stop, double-stop, dataclass defaults) stay
   green.

Gate: `pytest -q`, `flake8 src tests examples --max-line-length=100
--ignore=E203,W503`, `mypy src/pits_mras --ignore-missing-imports`.
