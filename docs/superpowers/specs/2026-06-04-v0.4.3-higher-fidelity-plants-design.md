# Higher-fidelity example plants (design spec)

**Goal:** Replace the three examples' toy *linear* surrogate plant steps with
physically-faithful **nonlinear** plant models, so the demos show the controller
handling genuine nonlinear dynamics (model-mismatch robustness — the point of
adaptive control). Release v0.4.3.

**Date:** 2026-06-04
**Sub-project:** 4 of the remaining v0.4.0 set.

## Current state

Each example advances its plant with an inline *linear* second-order step:
- manipulator: `ẍ = u − 4x − 4ẋ`
- AV: `ÿ = u − 2y − 3ẏ + gust`
- HVAC: `ẍ = u − 3x − 2ẋ`

The controller (LQR/CBF designed on the linear reference model) tracks these. The
README already (honestly) flags these as toy linear surrogates.

## Design

A new shared module **`examples/plants.py`** with three nonlinear, pure step
functions `f(state: Tensor[2], u: float, dt: float, **params) -> Tensor[2]`
(semi-implicit Euler). Each is designed so its **small-signal linearization
recovers the example's existing linear surrogate** — so the existing LQR/CBF
controller stays stabilizing by construction (verified: all three closed loops
stay bounded/finite under the reference-model LQR gain), while the nonlinearity
adds fidelity away from the operating point.

1. **`pendulum_step`** (manipulator joint, `m·l²=1`):
   `θ̈ = u − (g/l)·sin θ − b·θ̇`, defaults `g/l=4`, `b=4`.
   Linearizes (sin θ ≈ θ) to `u − 4θ − 4θ̇` = the toy. Gravity `sin` nonlinearity;
   `θ=0` (hanging) is the stable equilibrium. Reference model **unchanged**.
2. **`lateral_tyre_step`** (single-track lateral dynamics with tyre-force
   saturation): `ÿ = u − k·tanh(y) − c·ẏ + gust`, defaults `k=2`, `c=3`.
   Linearizes (tanh y ≈ y) to `u − 2y − 3ẏ + gust` = the toy; the lateral
   restoring **saturates** (`|k·tanh| ≤ k`) like a real tyre. Reference model
   **unchanged**.
3. **`rc_thermal_step`** (2-node RC building-thermal network, deviation-from-
   ambient coords) with a **saturated heater**:
   `Ṫ_z = a_zm(T_m−T_z) − a_za·T_z + b·sat(u)`, `Ṫ_m = a_mz(T_z−T_m)`,
   defaults `a_zm=2`, `a_za=1`, `a_mz=1`, `b=3`, `u_max=5`. State `[T_z, T_m]`
   (zone, thermal-mass temperatures). The RC network is linear + Hurwitz
   (eigenvalues ≈ −3.73, −0.27); the **only** nonlinearity is heater saturation.
   The HVAC example's **reference model is updated** to the RC linearization
   `A_m = [[−(a_zm+a_za), a_zm], [a_mz, −a_mz]]`, `B = [[b], [0]]` so the LQR
   tracks the zone temperature (verified stable, tracks the setpoint).

### Integration

Each example's `run()` imports the matching plant from `examples/plants.py`
(`sys.path.insert(parent)` + `from plants import …`, inside `run()` to match the
existing import-safe pattern) and uses it for the plant advance, replacing the
inline linear step. The controller / `RealtimeInferenceEngine` / metrics / figure
code is unchanged. Only the HVAC example additionally swaps its reference-model
matrices to the RC ones above. The plant state stays 2-D (matches `plant_dim=2`).

## Testing (TDD)

New `tests/test_example_plants.py` (imports `examples/plants.py` via the same
path trick) — physics sanity per plant:
- **pendulum:** rest `(0,0), u=0 → stays 0`; a small displacement with damping
  decays; `sin` nonlinearity: restoring at `θ=1.0` is `< ` the linear `4·θ`
  (saturating). Finite outputs.
- **lateral:** rest stays; tanh saturation: restoring magnitude `≤ k` for large
  `|y|`; gust shifts `ẏ`. Finite.
- **rc_thermal:** heater saturation: `u` beyond `u_max` gives the same `Ṫ_z` as
  `u_max`; with `u=0` the state relaxes toward `0` (dissipative); zone–mass
  coupling moves `T_m` toward `T_z`. Finite.

Existing `tests/test_examples.py` `run()` smoke tests must stay green (each demo
now uses a nonlinear plant but still returns finite metrics + a `Figure`; closed
loop stays stable — verified for all three). Gate: `pytest -q`, `flake8 src tests
examples --max-line-length=100 --ignore=E203,W503`, `mypy src/pits_mras
--ignore-missing-imports` (mypy targets the package, not `examples/`, but flake8
covers `examples/`).
