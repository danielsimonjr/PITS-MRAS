# v0.3.1 — Minimize / Simplify / Optimize (design spec)

**Goal:** A behavior- and API-preserving simplification + optimization pass.
Release v0.3.1; full TDD/dev-workflow; the test suite stays green throughout;
stale documents updated.

**Hard constraint (the contract):** every change is **output-identical**. No
public-API or `PITSMRASConfig` changes. The existing suite must stay green,
unchanged except where a test asserted a now-removed *internal* dict key. New
equivalence tests lock the refactors.

**Approach:** #2 (Measured) — A + C + *provably-equivalent* D.

## In scope

### A — Consolidate the quadratic-basis convention (DRY)

The upper-triangular `eᵢeⱼ` basis (diagonal coeff `= P[i,i]`, off-diagonal coeff
of `eᵢeⱼ` `= P[i,j] + P[j,i]`) is independently re-encoded in three files:

- `utils/lyapunov.py::quadratic_basis(e)` — the feature map `φ(e)`.
- `models/critic.py::QuadraticCritic.extract_P` / `.set_P` — matrix ↔ `W_c` weights.
- `training/irl_trainer.py::_quadratic_features` / `_vec_to_P` — features + matrix.

**Design:** make `utils/lyapunov.py` the single source of truth. Add:

- `pack_symmetric(P: Tensor) -> Tensor` — symmetric `[n, n]` → basis-coefficient
  vector `[n(n+1)/2]` (diag = `P[i,i]`, off-diag = `P[i,j] + P[j,i]`).
- `unpack_symmetric(vec: Tensor, n: int) -> Tensor` — inverse (off-diag split in
  half across `(i,j)`/`(j,i)`), yielding a symmetric matrix.

`quadratic_basis` stays (already canonical). Rewrite `extract_P`→`unpack_symmetric`,
`set_P`→`pack_symmetric`, `_quadratic_features`→`quadratic_basis`,
`_vec_to_P`→`unpack_symmetric`. Delete the three hand-rolled loops.

**Behaviour guarantee:** identical convention; existing `set_P`/`extract_P`
round-trip tests, the costate tests, and `train_irl_critic` convergence tests
stay green. New test: `eᵀPe == quadratic_basis(e) · pack_symmetric(P)` and
`unpack_symmetric(pack_symmetric(P)) == P` for random symmetric `P`.

### C — Collapse the PITNN `forward` dual dict keys

`models/pitnn.py::PITNN.forward` returns both "brief" keys (`f`, `H`) and "spec"
keys (`f_hat`, `H_val`); `f == f_hat` and `H == H_val` exactly. The `f`/`H`
aliases are consumed **only** by `tests/test_models.py` (production uses
`f_hat`/`energy_loss`/`attn_reg_loss`/`context`).

**Design:** drop the two exact-duplicate keys `f` and `H`. Keep `f_hat`,
`H_val`, `context`, `alpha`, `h_enc`, `P_diss`, `energy_loss`, `attn_reg_loss`.
Update the one test (`test_models.py`) to the canonical keys (`f`→`f_hat`,
`H`→`H_val`). Update the `forward` docstring/return type note.

**Behaviour guarantee:** every production consumer already uses the retained
keys; only the one test changes.

### D — Measured performance (profile first; only output-identical opts)

Profile the hot paths (`pytest`-level timing of `KKTProjectionLayer.forward`,
`RealtimeInferenceEngine.step`, `cotraining_loop` step). Apply only changes a
before/after `torch.allclose` proves identical:

- **D1.** `KKTProjectionLayer.forward` recomputes `_constraints_and_jac` +
  `_assemble_J` at `z_star` for the implicit-diff one-step, after the Newton
  loop already computed them at the final iterate — reuse them. (Output-identical
  if the final iterate == `z_star`.)
- **D2.** `extract_P`/`set_P` use `pack/unpack_symmetric` (from A) implemented
  vectorized (no Python `for` loops), dropping per-call loop overhead in
  `cotrain`'s per-step `critic_convergence`.
- **D3.** Avoid redundant `extract_P` recomputation per cotrain step where the
  same `P̂` is already available.

Each D-item is independently gated; any that is not provably output-identical is
dropped (not forced).
